# Workspace Main Layout Design

## 背景

Refinex Wiki 已经有基于 Plate 的编辑器页面，当前 `app/editor/page.tsx` 直接渲染 `PlateEditor`。桌面化后，产品主界面需要从单编辑器页面升级为 Notion 式文档工作区：左侧管理本地 Markdown 工作区，中间编辑当前文档，右侧承载 AI 辅助入口。

本设计只覆盖第一版主界面布局和工作区交互，不覆盖完整 AI 调用、全文索引、同步、多窗口、Git 集成或发布更新。

## 目标

第一版主界面采用三列布局：

- 左列：工作区搜索和目录文档树，可折叠。
- 中列：主编辑器区域，保持文档编辑沉浸感。
- 右列：AI 面板，可折叠，默认折叠。

第一版必须接入真实本地工作区：首次进入选择本地文件夹，之后恢复最近工作区；目录树展示 `.md` 和 `.mdx` 文件；搜索支持文件名、路径和 Markdown 一级标题。

## 非目标

- 不做全文搜索或内容片段高亮。
- 不接入真实 AI 对话调用。
- 不实现多工作区管理。
- 不实现文件拖拽重排、重命名、删除、新建文件等文件管理能力。
- 不改造 Plate 编辑器核心插件架构。
- 不设计移动端优先体验；桌面端是主目标。

## 布局结构

采用组件化 Workspace Shell，而不是把侧栏和 AI 面板塞进 `PlateEditor` 内部。

推荐组件边界：

- `WorkspaceLayout`：三列整体布局、折叠状态、列宽约束。
- `WorkspaceSidebar`：左侧容器，负责搜索区、目录树区、工作区状态区。
- `WorkspaceSearch`：输入搜索词，展示匹配范围说明和空结果状态。
- `DocumentTree`：展示目录和 Markdown 文件，支持文件夹展开/折叠。
- `EditorPane`：中间区域，负责顶部路径栏和编辑器容器。
- `AiSidePanel`：右侧 AI 面板，默认折叠，展开后显示占位结构。

`app/editor/page.tsx` 只负责装配 Workspace Shell 和页面级 Toaster。

## 默认尺寸

- 左侧栏默认宽度：`280px`。
- 左侧栏折叠宽度：`48px`。
- 右侧 AI 面板默认折叠宽度：`48px`。
- 右侧 AI 面板展开宽度：约 `340px`。
- 编辑器正文最大宽度：约 `820px`。
- 主界面高度：占满视口。

列宽需要使用稳定尺寸，折叠/展开不能导致编辑器内部工具栏发生明显错位。

## 左侧工作区

左侧栏包含：

- 顶部工作区名称和折叠按钮。
- 搜索输入框，占位文案为“搜索标题或路径”。
- 目录文档树。
- 底部最近工作区状态或切换入口。

首次没有最近工作区时，左侧显示选择文件夹入口。用户选择文件夹后，系统读取目录树并持久化最近工作区路径。后续进入页面时自动恢复最近工作区；如果路径不可访问，则显示恢复失败状态，并允许重新选择文件夹。

目录树规则：

- 只展示文件夹、`.md`、`.mdx` 文件。
- 文件夹支持展开和折叠。
- 当前文档高亮。
- 文件按文件夹优先、文件其次排序；同层按名称排序。
- 暂不展示非 Markdown 文件。

## 搜索

第一版搜索范围：

- 文件名。
- 相对路径。
- Markdown 一级标题。

搜索行为：

- 输入搜索词后过滤目录树或展示扁平搜索结果，两者实现时可择一，但交互必须保持明确。
- 匹配结果点击后打开对应文档。
- 空结果显示“没有匹配的文档”。
- 搜索不读取全文内容，不展示正文命中片段。

一级标题解析规则：

- 读取文档开头附近第一个 `# ` 标题作为标题。
- 如果没有一级标题，使用文件名作为标题。
- 标题解析失败不阻断目录树展示。

## 编辑器区域

中间区域包含：

- 顶部路径栏：显示当前工作区内的相对路径。
- 文档编辑器：复用现有 `PlateEditor` 能力。
- 空状态：未选择文档时，显示选择文档或打开工作区的提示。

第一版不强制实现 Markdown 文件内容加载到编辑器；如果实现阶段选择接入文档读取，也只能通过最小适配层传入 `PlateEditor`，不能把 Workspace Shell 状态写进 Plate 插件内部。

如果接入真实文档内容，读取失败时中间区域展示错误状态，不应让整个页面崩溃。

## AI 面板

右侧 AI 面板默认折叠，只显示窄栏图标。点击后展开为约 `340px` 宽。

展开内容第一版只做占位：

- 当前文档名称或路径。
- 快捷动作入口：总结此页面、解释选中内容、生成大纲。
- 输入框占位文案。
- 明确的未接入状态，不调用 `/api/ai/*`。

AI 面板折叠状态应能被页面状态保存；是否跨会话持久化可在实现阶段根据成本决定，但同一次页面会话内必须稳定。

## 数据流

前端通过 Workspace Shell 管理 UI 状态：

- 当前工作区路径。
- 当前文档路径。
- 文档树数据。
- 搜索词和搜索结果。
- 左侧栏折叠状态。
- AI 面板折叠状态。

桌面端文件系统能力通过 Tauri 提供：

- 选择目录。
- 读取目录树。
- 读取 Markdown 文件标题。
- 保存和恢复最近工作区路径。

Web 环境下可以保留降级占位，避免非 Tauri 环境直接崩溃。

## 错误处理

- 没有工作区：显示选择文件夹入口。
- 最近工作区不可访问：显示恢复失败，并允许重新选择。
- 目录读取失败：显示错误说明和重试入口。
- 文档读取失败：编辑器区域显示错误状态。
- 搜索解析失败：跳过失败文件，不阻断其他结果。

错误文案应简短、面向用户，不暴露底层堆栈。

## 测试策略

第一版需要覆盖：

- `WorkspaceLayout` 折叠/展开状态。
- `DocumentTree` 文件夹展开、当前文档高亮、Markdown 文件过滤。
- `WorkspaceSearch` 文件名、路径、一级标题搜索。
- 最近工作区恢复成功和失败状态。
- AI 面板默认折叠和展开占位。
- 非 Tauri 环境下的降级空状态。

如果暂时没有完整测试框架，至少在实现计划中补充 focused component tests 和手动验证命令。

## 已确认决策

- 视觉方向：Notion 工作区式。
- 第一版范围：可用工作区雏形。
- 工作区来源：首次选择文件夹，之后恢复最近工作区。
- 搜索范围：文件名、路径、Markdown 一级标题。
- AI 面板：默认折叠，展开后为占位，不调用 AI。
- 实现方案：组件化 Workspace Shell。
